设计模式一共分三大类
1.建造模式
单例模式:用在复杂对象的创建场景中，该对象全局只有一个。
工厂模式:用于生产对象的场景中
抽象工厂:同工厂模式，只不过抽象工厂中工厂也是需要实现的。
build模式:对于一个对象属性特别多，构造函数特别多的场景中使用。
prototype(原型)模式:用于基于一个对象copy一个副本的场景中。
2.结构模式
适配模式:分三种类适配、对象适配、接口适配主要用于将一种类、对象、接口适配到另一个接口中去。
代理模式:这个好理解，其实spring的aop核心就是采用了动态代理模式。
桥接模式:想想jdbc的DriverManager，这个的实现就是采用桥接模式
装饰器模式:想想jdk中的IO流，就是采用该模式实现。
门面模式:多个对象组合的时候使用，减少各个对象之间的耦合，想想cpu、memory、dis和computer的关系
享元模式:想想对象池，数据库连接池，就是采用该模式实现
